# PLANO DE IMPLEMENTAÇÃO: PRODUTOS DINÂMICOS NO BANCO DE DADOS

Este documento descreve o passo a passo de como migraremos o sistema atual (produtos estáticos no HTML) para um sistema dinâmico, onde os produtos são salvos no banco de dados SQLite, servidos pela API Node.js e renderizados pelo JavaScript no Frontend.

---

## FASE 1: Banco de Dados (Backend - src/js/database.js)

Atualmente, o banco de dados apenas cria a tabela `users`. Precisaremos instruir o banco a criar também uma tabela `products`.

O que será feito:
1. Abrir o arquivo `src/js/database.js`.
2. Adicionar o script de criação da tabela de produtos (caso ela não exista).
3. (Opcional) Adicionar alguns produtos "semente" (Seed) direto no código para que a loja não fique vazia na primeira vez que rodar.

Estrutura da Tabela `products`:
- `id` (INTEGER, Primary Key)
- `name` (TEXT) - Ex: "Tênis Nike Revolution 6"
- `category` (TEXT) - Ex: "Tênis"
- `price` (REAL) - Ex: 100.00
- `old_price` (REAL) - Ex: 200.00
- `image_url` (TEXT) - Ex: "src/assets/tenis.png"
- `description` (TEXT)

---

## FASE 2: Criação de Rotas na API (Backend - src/js/server.js)

O servidor precisa de portas de acesso (rotas) para que o frontend consiga consultar os produtos.

O que será feito:
1. Criar a rota `GET /api/products`:
   - Essa rota fará um `SELECT * FROM products` no banco de dados e retornará uma lista (JSON) de todos os produtos cadastrados.
2. (Opcional) Criar a rota `GET /api/products/:id`:
   - Para consultar um único produto (útil para a página de detalhes do produto).
3. (Futuro) Criar a rota `POST /api/products`:
   - Uma rota protegida (apenas para administradores) para cadastrar novos produtos via sistema, sem precisar mexer no código.

---

## FASE 3: Consumo dos Dados (Frontend - JavaScript)

Hoje, os produtos estão fixos (`hardcoded`) no `Homepage.html` e `ProductListdesign.html`. Precisaremos remover o código repetitivo e substituí-lo por JavaScript que gera o HTML dinamicamente.

O que será feito:
1. Criar um novo arquivo `src/js/utils/productService.js`:
   - Conterá funções como `fetchProducts()` que usam o comando `fetch('http://localhost:3001/api/products')` para pegar os dados da API.
2. Alterar o Frontend (ex: `main.js` ou um novo componente):
   - Ao carregar a página (DOMContentLoaded), o JS vai chamar a API.
   - O JS fará um laço de repetição (`forEach`) nos produtos recebidos.
   - Para cada produto, o JS montará aquele bloco HTML de "card de produto" e o injetará na tela (usando `document.getElementById('secao-produtos').innerHTML`).

---

## FASE 4: Limpeza do HTML

O que será feito:
1. Ir em `Homepage.html`, `ProductListdesign.html`, etc.
2. Apagar todos os blocos de produtos repetidos (as divs `<div class="div-products-line-item">`).
3. Deixar apenas uma div vazia ou um contêiner (ex: `<div id="produtos-container"></div>`), que servirá como a "caixa" onde o JavaScript colocará os produtos que vierem do banco de dados.

---

## BENEFÍCIOS DA MUDANÇA
- **Escalabilidade:** Adicionar 1 ou 1000 produtos dará o mesmo trabalho.
- **Manutenção:** Se precisar mudar o layout do card de produto, você muda em apenas 1 lugar (no JavaScript) e não em 50 blocos HTML diferentes.
- **Painel Administrativo:** Abre portas para criarmos uma tela futuramente onde você cadastra, edita e deleta produtos preenchendo um formulário no próprio site, sem tocar em código.

---

## FASE 5: Correções Visuais Pendentes (Layout e Favicon)

### 1. Correção do Layout (Descrição e Preço fora do quadrado)
No `Homepage.html`, o texto e os preços do produto estão sendo renderizados fora do bloco branco porque a div `.div-products-line-item2` foi colocada no HTML após o fechamento da div `.div-products-line-item1` (que desenha o quadrado branco e a sombra).

**Opções de Resolução:**
*   **Solução A (Alterar o HTML):** Mover toda a estrutura do texto e preço (`<div class="div-products-line-item2">...</div>`) para *dentro* da `<div class="div-products-line-item1">`, colocando-a antes da tag de fechamento `</div>`. Dessa forma, todos os elementos ficarão contidos no mesmo quadrado branco.
*   **Solução B (Alterar o CSS):** Se a intenção for manter as divs `.div-products-line-item1` (imagem) e `.div-products-line-item2` (textos) separadas, deve-se acessar o `Homepage.css` e transferir as propriedades de estilo (fundo branco, padding, sombra e transição de hover) da classe `.div-products-line-item1` para a classe pai (contêiner principal) `.div-products-line-item`.

### 2. Correção do Ícone da Aba (Favicon) quebrando nas outras páginas
O problema ocorre devido ao "caminho relativo" informado no atributo `href`. O caminho base não é o mesmo para páginas na raiz (`Homepage.html`) e páginas dentro da pasta `src/pages/`.

**Como resolver:**
*   **Em `Homepage.html` (raiz):** O caminho correto não deve usar `../`, pois não há pasta anterior para voltar. O correto é:
    `<link rel="icon" href="./src/assets/logo_digital_store.svg" type="image/png">`
*   **Nos arquivos dentro de `src/pages/` (ex: `cart.html`, `Page-Producto.html`):** O caminho deve voltar uma pasta para acessar a raiz do `src` e depois ir para `assets`. O correto é:
    `<link rel="icon" href="../assets/logo_digital_store.svg" type="image/png">`
    *(Obs: se já estiver assim e o ícone não atualizar, é necessário limpar o cache do navegador com Ctrl + F5, pois favicons costumam ficar retidos no cache).*
